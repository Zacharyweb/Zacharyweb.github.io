http协议浅尝

发表于 2016-12-21 | 分类于 后端开发 | 点击量 次

http协议是客户端和服务器通信的一种规范、或者说是一种语言，本文主要记录一些自己对http协议的理解。

从http角度分析网页加载过程

当你输入一个网址到网页加载呈现在你眼前，这个看似简答的过程中其实发生了如下步骤：

  1. 浏览器会把输入的域名发送给DNS服务器，通过DNS服务器解析为IP地址。
  2. IP地址是具体的用来定义网络中的一台计算机中，通过IP地址访问到需要访问的服务器。
  3. 通过IP地址找到服务器之后，再通过端口来找服务器进程通信。
  4. 服务器收到请求后返回相应的数据给客户端浏览器。
  5. 浏览器根据服务器返回的数据对页面进行渲染并呈现。

TCP三次握手连接

在客户端跟服务器进行数据交互时，其实要先通过TCP三次握手连接确定已经建立连接。

  • 第一次：客户端向服务器发送消息，发完之后，客户端不知道服务器有没有收到这个消息。如果收到服务器给的消息之后就意味着服务器可以正常的接收我的消息了。

  • 第二次：服务器接收到了客户端发来的消息之后，给客户端回复一个消息。

  • 第三次：客户端接收到服务器给的消息之后再次给服务器端发送一段消息

经历这三次之后，就意味着：客户端可以确定服务器可以收到自己的消息，服务器也可以确定客户端可以收到自己的消息。

http协议相关概念

url

例如http://example.com/part/index.htm 就是一个url，用于定位一个网络资源。

  • http:// 协议类型
  • example.com 域名，主机名
  • /part/index.htm 请求路径

域名

例如www.taobao.com就是一个域名，通常我们可以通过域名访问一个网站，但其实域名本身是无法实现通信的，域名的作用仅仅是为了方便记忆。每一个域名都对应一个唯一的IP地址，浏览器通过域名从DNS服务器获取相应的IP地址后才能向正确的服务器发送请求，来获得数据，实现通信。

IP地址

IP地址实际上就是用来定位一台计算机的位置的，浏览器通过IP地址定位到了服务器。那么定位到计算机之后，到底要和谁通信？

端口

IP地址是家、端口号就是门，具体的定义一台计算机中的程序的。

端口号

用来通信的软件才会占用一个端口号，端口号不能重复。

http的默认端口就是80，如果你输入了www.taobao.com www.taobao.com:80
